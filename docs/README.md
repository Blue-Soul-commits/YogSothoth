# 文档总览

本目录是「MCP 多仓库代码助手」项目的文档中心，主要面向两类读者：

- 开发者：需要了解系统架构、目录结构、接口设计、如何本地开发与调试。
- 使用方（包括大模型 / 集成方）：需要了解 MCP 工具的能力与调用方式。

推荐阅读顺序如下。

---

## 1. 项目概览与规划

这些文件在仓库根目录下：

- [MCP_PROJECT_OVERVIEW.md](../MCP_PROJECT_OVERVIEW.md)  
  系统目标、整体架构、各角色边界、MCP 暴露的工具能力说明。
- [MCP_PROJECT_PLAN.md](../MCP_PROJECT_PLAN.md)  
  里程碑拆分、任务列表、验收标准，指导后续迭代开发。

如果你是第一次接触本项目，先完整读完这两份文档。

---

## 2. 代码结构

- [project-structure.md](./project-structure.md)  
  给出后端 / 前端 / 文档目录的推荐结构，把「Core Library / HTTP 管理 API / MCP Server / 前端」如何落到文件系统中说清楚，便于统一规范。

在开始编码前，建议根据本项目的实际情况，对该文档中的结构进行微调并最终定稿。

---
